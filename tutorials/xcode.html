Installing and Using Xcode Templates

Included within AudioKit are Xcode templates that will help you get started when creating AudioKit classes from scratch. The templates are for Objective-C or Swift and include a "conductor" class for managing audio, an "instrument" class for designing instruments and their associated notes, and a "processor" class, which is a special case of an instrument that process audio created by other instruments.

To install the templates, make sure you have the Xcode folder for user file templates and if not, create it:

{% highlight bash %} mkdir -p ~/Library/Developer/Xcode/Templates/File\ Templates/ {% endhighlight %}

Then, copy the "Templates / File Templates / AudioKit" folder from your downloaded AudioKit repository (in this case, AudioKit was installed in ~/Developer)

{% highlight bash %} cp -r ~/Developer/AudioKit/Templates/File\ Templates/AudioKit ~/Library/Developer/Xcode/Templates/File\ Templates/ {% endhighlight %}

Now, when you choose "File / New / File..." in Xcode, the templates will appear in an AudioKit section of the dialog box:

Installing Xcode Documentation

Code completion, pop-ups, and sidebar information are automatically created in Xcode, but if you would like Apple-style API reference documentation, download the docset:

Un-tar it and place it in the Shared Documentation folder:

{% highlight bash %} tar -xf ~/Downloads/io.audiokit.AudioKit.docset.tar.gz -C ~/Library/Developer/Shared/Documentation/DocSets/ {% endhighlight %}
