FeatureV2.xV3 Beta
Live Coding with Playgrounds 35 Objective-C Playgrounds based off of KZPlayground are powerful but do require extending Xcode with dynamic code injection. 40+ Standard Xcode Swift Playgrounds with some limitations such as less user interface control and no live-input processing on iOS and limited AKNode support on OSX. Hopefully this will improve over time.
Graphing Integrated EZAudio graphs of waveforms, FFT spectrum etc. Pretty badass. No graphing, yet.
Audio Operations 120 57 and counting
Audio Engine Csound AVAudioEngine, Soundpipe, which includes code from Chuck, Csound, Faust, Guitarix, ToneStack. Basically, if is open-sourced we can include it. Email us!
Integration with Other Audio Tools Very little integration with other audio tools is possible. Audio comes in from the microphone and out of the speakers and any audio has to come from the microphone, the file system, iTunes, or generated in AudioKit. This version attempts to be a cooperative member of any audio ecosystem. AudioKit's nodes are just Audio Units themselves, so they can be dropped in to a signal chain as you wish. Any third party tool can be easily included in the signal chain as well.
Language Written in Objective-C, but works just as well with a Swift-based app. Written in Swift with C/C++ underpinnings. Probably works fine with Objective-C, but we're focussing on Swift from now on.
Installation Fairly easy installation as a static or dynamic library. Dependencies include libCsound and libSndFile. CocoaPods Support. Just drag the framework into your project. Planned CocoaPods and Carthage support.
Docs Gorgeous docs generated by AppleDoc. Online tutorials and videos. Gorgeous docs generated by Jazzy. Playgrounds do a great job of replacing tutorials.
License MIT License, but you need to use the dynamic library. MIT License. Period.
Examples Included examples for iOS, OSX, and tvOS, plus a repository of example projects. None yet, but perhaps this is an opportunity for you to contribute!
Templates Xcode templates for common classes. No templates. While we're able to write them, they are not too high on our priorit list, so we'll include the task of creating templates to the first-timers list.
Tests Extremely well tested and under Travis CI. Test infrastructure started, but not many tests written yet. This is still very important to us and it is a high priority.
Sensible Defaults and Presets Most operations have sensible defaults and presets. Defaults, yes. Presets not so much. But they are not off the radard and they will be added.
Stability This code has been stable for a while and is powering apps in the Apple Store. Beta software. It's called bleeding edge for a reason.
