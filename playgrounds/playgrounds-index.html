Playgrounds will rock your world

Playgrounds are a means to develop audio applications with immediate aural and visual feedback about what you are creating. Gone are the days of write-compile-listen-and-rewrite. Not only can you develop quickly, but you can stumble upon amazing sounds along the way. Truly, you're learning to play an instrument, and that instrument is AudioKit code. As you can tell, we're super excited about how this will help developers create amazing things.

Installation and First Run

The Playground.m file

This will be your main workspace while developing playgrounds (though you can import other classes too). There are two methods that need to be implemented:

setup method

This method run exactly once when the playground is first run. You can add instruments here if you know they will not need to be changed during the run. You can also setup standard UI elements here.

run method

This method is run every time and will be where you do your experimenting. A couple of things to be careful about are to make sure you declare instruments with a number tag using "instrumentWithNumber" or "initWithNumber" so that the instrument is replaced on each run. Also, instead of "play"ing instrument, you will want to "restart" it so that any previous sounds that had been started are stopped rather than layered onto one another.

Playground helper methods

UI Helper MethodDescription
addButtonWithTitle: block: Add a button to the timeline that executes the given code
addSliderForProperty: title: Add a slider to the timeline for a given property
addRepeatSliderForInstrument: Add a slider control for spawning recurring phrases
Plot Helper MethodDescription
addAudioInputPlot: Add a waveform representation of the audio input, usually the microphone
addAudioOutputPlot: Add a monophonic audio output plot
addStereoOutputPlot: Add a stereo audio plot with the left channel on top and right on the bottom
addAudioInputFFTPlot Add a FFT of the audio input
addAudioOutputFFTPlot Add a FFT of the audio output
addPlotForInstrumentProperty: withLabel: Add a plot of a given instrument property
addFloatPlot:withLabel: Add a plot of any floating point number
addTablePlot: Add a table plot to the timeline
addAudioInputRollingWaveformPlot Add a plot of the input rolling waveform
addAudioOutputRollingWaveformPlot Add a plot of the output rolling waveform

Installation Instructions

Playgrounds require a couple of programs to be installed on your computer in addition to AudioKit. We have built an installation script so all you have to do is go into the Playgrounds folder and run install.sh:

{% highlight bash %} $ cd Playgrounds $ ./install.sh {% endhighlight %}

This will install the Xcode Dynamic Code Injection plugin and the GEM for a file-change watcher called Kicker.

NOTE: If you update Xcode at all, you'll need to run the installation script again so that the Dynamic Code Injection plugin and Kicker are reinstalled.

Running a Playground

The best way to run the playground is to use the included run.sh script. This will start an interactive bash session asking you what you would like to do. On your first run you will want to choose option 3 and install one of the default playgrounds. In the future when you're developing, you will want to run with option 1, the current playground. Be careful not to accidentally trash your existing playground by installing a default on top of it!

{% highlight bash %} ┌─ ~/Developer/AudioKit/Playgrounds └─➤ ./run.sh 1) Run Current Playground 3) Choose Playground from List 2) Run Empty Playground Enter your choice: 3 1) BambooSticks 6) Oscillator 11) Table 2) Default 7) PluckedString 12) Tambourine 3) Mandolin 8) Sequencing 13) VCOscillator 4) Marimba 9) Sleighbells 14) Vibraphone 5) MicrophoneAnalysis 10) StruckMetalBar Enter the number of the playground you want: {% endhighlight %}

Setting up a playground from scratch

  1. Install cocoapods if you haven't already
  2. Create a new iOS project for your playground
  3. Close the project, and from the project folder, run pod init
  4. Edit the podfile to include KZPlayground like in the AudioKitPlayground project
  5. Run kicker -sql 0.05 YourProjectFolder replacing YourProjectFolder with folder containing your watched classes
  6. Create a new playground class that is a subclass of AKPlayground
  7. Edit the playground class's header file: {% highlight objective-c %} #import "AKPlayground.h" @interface Playground : AKPlayground @end {% endhighlight %}
  8. Create the setup and run methods in the implementation file.
