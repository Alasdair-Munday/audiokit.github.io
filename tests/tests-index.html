Continuous Integration Testing

We've recently synced the AudioKit GitHub repository with TravisCI. Travis ensures that the test suite is run and that included examples compile on a fresh system. AudioKit has some included examples, but there are many more in the "Examples" repository, which is also tested for build succeess on each project with Travis CI. We also happen to run a test on this web site with TravisCI ensuring we have no broken links.

Command Line Tests

You can run our tests from the command line to not only test a particular operation, but also to very quickly hear what something sounds like without building a project or running a playground.

We use the XCPretty gem to condense test output. To install xcpretty, use:

{% highlight bash %} $ gem install xcpretty {% endhighlight %}

To build all the tests, first change directories in to the tests folder and run the build_all.sh script:

{% highlight bash %} $ cd Tests $ ./build_all.sh {% endhighlight %}

Then, you can either run them all or be prompted to run just one using the run.sh script:

{% highlight bash %} $ ./run.sh 1) Run All Tests 2) Choose Test from List Enter your choice: {% endhighlight %}

To run the continuous integration tests locally, change your directory to the top level AudioKit directory and run:

{% highlight bash %} > ./Tests/travis.sh {% endhighlight %}

This will run tests and build projects and if all goes well the last line should read: {% highlight bash %} AudioKit passed validation. {% endhighlight %}
