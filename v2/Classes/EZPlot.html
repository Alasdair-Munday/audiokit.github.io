Overview

EZPlot is a cross-platform (iOS and OSX) class used to subclass the default view type (either UIView or NSView, respectively). ## Subclassing Notes This class isn't meant to be directly used in practice, but instead establishes the default properties and behaviors subclasses should obey to provide consistent behavior accross multiple types of graphs (i.e. set background color, plot type, should fill in, etc.). Subclasses should make use of the inherited properties from this class to allow all child plots to benefit from the same

Properties

color

The default color of the plot's data (i.e. waveform, y-axis values). For iOS the color is specified as a UIColor while for OSX the color is an NSColor. The default value on both platforms is red.
{% highlight objective-c %}@property (nonatomic, strong) IBInspectable UIColor *color{% endhighlight %}

gain

The plot's gain value, which controls the scale of the y-axis values. The default value of the gain is 1.0f and should always be greater than 0.0f.
{% highlight objective-c %}@property (nonatomic, assign) IBInspectable float gain{% endhighlight %}

plotType

The type of plot as specified by the EZPlotType enumeration (i.e. a buffer or rolling plot type).
{% highlight objective-c %}@property (nonatomic, assign) EZPlotType plotType{% endhighlight %}

shouldFill

A boolean indicating whether or not to fill in the graph. A value of YES will make a filled graph (filling in the space between the x-axis and the y-value), while a value of NO will create a stroked graph (connecting the points along the y-axis).
{% highlight objective-c %}@property (nonatomic, assign) IBInspectable BOOL shouldFill{% endhighlight %}

shouldMirror

A boolean indicating whether the graph should be rotated along the x-axis to give a mirrored reflection. This is typical for audio plots to produce the classic waveform look. A value of YES will produce a mirrored reflection of the y-values about the x-axis, while a value of NO will only plot the y-values.
{% highlight objective-c %}@property (nonatomic, assign) IBInspectable BOOL shouldMirror{% endhighlight %}

Instance Methods

clear

Clears all data from the audio plot (includes both EZPlotTypeBuffer and EZPlotTypeRolling)
{% highlight objective-c %}- (void)clear{% endhighlight %}

updateBuffer:withBufferSize:

Updates the plot with the new buffer data and tells the view to redraw itself. Caller will provide a float array with the values they expect to see on the y-axis. The plot will internally handle mapping the x-axis and y-axis to the current view port, any interpolation for fills effects, and mirroring.
{% highlight objective-c %}- (void)updateBuffer:(const float *)buffer withBufferSize:(UInt32)bufferSize{% endhighlight %}
  • buffer: A float array of values to map to the y-axis.
  • bufferSize: The size of the float array that will be mapped to the y-axis.

Discussion

> %warning% > Warning: The bufferSize is expected to be the same, constant value once initial triggered. For plots using OpenGL a vertex buffer object will be allocated with a maximum buffersize of (2 * the initial given buffer size) to account for any interpolation necessary for filling in the graph. Updates use the glBufferSubData(...) function, which will crash if the buffersize exceeds the initial maximum allocated size.
